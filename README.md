# 🚀 A/B Testing

## 📌 О проекте

Данный репозиторий представляет собой комплексное исследование в области продуктовой аналитики и А/В-тестирования. Проект состоит из двух логических частей:

1. **Симуляционный фреймворк:** Валидация статистических критериев (A/A и A/B тесты) с нуля, оценка их корректности на различных распределениях данных и расчет мощности тестов.
2. **Управление дисперсией на реальных данных:** Применение продвинутых статистических и ML-алгоритмов (CUPED, Iterative Multi-CUPED, ML-CUPAC) для кратного снижения дисперсии целевых метрик финтех-продукта (премиум-сегмент). Это позволяет значительно сократить необходимое время проведения теста и сэкономить маркетинговый бюджет на закупку трафика.

## 🛠 Стек технологий

* **Язык и обработка данных:** Python (Pandas, NumPy)
* **Статистика и математика:** SciPy, Statsmodels
* **Machine Learning:** Scikit-Learn (`HistGradientBoostingRegressor`, `KFold`, `cross_val_predict`)
* **Визуализация:** Seaborn, Matplotlib
* **Методологии:** A/A-тестирование, T-tests, CUPED / Multi-CUPED / ML-CUPAC, Bootstrapping, OOF (Out-of-Fold) Cross-Validation, Монте-Карло симуляции.

## 🧠 Реализованный функционал и Математический аппарат

### Часть 1: Валидация критериев (Симуляции)

* **A/A тесты (Проверка FPR):** Симуляция тысяч экспериментов на однородных данных для проверки равномерности распределения p-value и удержания ошибки I рода (False Positive Rate) на уровне заданного $\alpha = 0.05$.
* **A/B тесты (Проверка TPR / Мощности):** Симуляция тестов с добавленным минимально детектируемым эффектом (MDE) для оценки реальной статистической мощности.
* **Влияние распределений:** Анализ поведения t-критерия на нормальных и скошенных (long-tail) распределениях, характерных для выручки и конверсий.

### Часть 2: Снижение дисперсии (Variance Reduction)

В А/Б-тестировании необходимый размер выборки $n$ линейно зависит от дисперсии метрики $\sigma^2$:


$$n \approx \frac{2\sigma^2 (Z_{1-\alpha/2} + Z_{1-\beta})^2}{MDE^2}$$


Для сокращения $n$ были реализованы следующие алгоритмы:

1. **Классический CUPED:** Использование одной доэкспериментальной ковариаты $X$ для коррекции целевой метрики $Y$. Гарантированное снижение дисперсии определяется квадратом корреляции $\rho^2$:

$$Y_{cuped} = Y - \theta (X - \mathbb{E}[X]), \quad \text{где } \theta = \frac{Cov(Y, X)}{Var(X)}$$


2. **Iterative Multi-CUPED:** Многомерное расширение. Итеративный подбор весов для топ-N ковариат, что позволяет бороться с мультиколлинеарностью признаков.
3. **ML-CUPED (CUPAC):** Применение Градиентного Бустинга (`HistGradientBoostingRegressor`) для поиска нелинейных связей. Алгоритм сворачивает всю историю пользователя в единый скор $Y_{pred}$. Для предотвращения утечки данных (Data Leakage) предсказания строятся строго через **5-fold Out-of-Fold (OOF)** кросс-валидацию.

## 📊 Описание данных и метрик

Проект применяет методы на реальном финтех-датасете (~83 000+ наблюдений). В анализе участвуют как целевые продуктовые метрики, так и исторические признаки (ковариаты).

### 🎯 Целевые метрики (Таргеты)

| Метрика | Тип | Описание |
| --- | --- | --- |
| `life_1_day_flg` | Бинарная | **Конверсия 1-го дня.** Флаг того, совершил ли клиент целевое действие (подключение услуги/покупка) в первый день эксперимента. |
| `life_32_day_flg` | Бинарная | **Долгосрочный Retention (32 дня).** Флаг продления подписки или сохранения активности на 32-й день (метрика с низким baseline и огромной дисперсией). |
| `orig_cost_sum` | Непрерывная | **Суммарные затраты/выручка.** Финансовая метрика, характеризующая LTV пользователя на горизонте тестирования (имеет сильный перекос вправо). |

### 🔍 Ключевые ковариаты (Исторические данные)

*Данные признаки собраны строго до момента разделения пользователей на группы А и Б, что делает их валидными для алгоритма CUPED.*

| Ковариата | Тип | Описание |
| --- | --- | --- |
| `loyalty_score` | Категориальная/Числовая | Внутренний скоринг лояльности клиента к банку/сервису. |
| `attrition_score` | Числовая | Предиктивный скор вероятности оттока клиента (Churn rate) до начала эксперимента. |
| `ag` (Age) | Числовая | Возраст клиента. |
| `Deneg_bolee_10tys_0` / `1` | Бинарная | Индикаторы (флаги) того, совершал ли клиент транзакции суммой свыше 10 000 рублей в историческом периоде. |

*(ML-модель автоматически отбирает и комбинирует данные фичи для максимизации корреляции с таргетом).*

## 📈 Главные результаты и Бизнес-эффект

### 1. Симуляционная часть

* Математически и визуально доказана **корректность работы t-критерия**: на однородных данных (А/А тесты) количество ложноположительных срабатываний не превышает заявленные 5%.
* Наглядно (через QQ-plots и KDE) показано, как "тяжелые хвосты" метрик снижают чувствительность тестов и требуют кратного увеличения трафика.

### 2. Прикладная аналитика (Реальные данные)

Применение ML-CUPAC на сегменте активных пользователей показало выдающиеся результаты по сжатию дисперсии (и, как следствие, сужению доверительных интервалов):

* **Метрика конверсии (`life_1_day_flg`):** Дисперсия снижена на **27.67%**.
* **Долгосрочное удержание (`life_32_day_flg`):** Дисперсия снижена на **21.34%**.
* **Финансовая метрика (`orig_cost_sum`):** Дисперсия снижена на **7.40%** (что для сильно скошенных денежных метрик является отличным результатом, позволяющим детектировать меньшие эффекты).

### 💰 Экономия для бизнеса (Traffic & CAC)

За счет снижения дисперсии на ~27.6%, требуемый объем выборки для фиксации MDE упал с **40 000** до **28 932** пользователей.

* **Ускорение теста (Time-to-Market):** Тест завершается в **1.38 раза быстрее** (экономия почти 3 дней).
* **Снижение затрат на трафик:** При средней стоимости привлечения клиента (CAC), алгоритм экономит бизнесу миллионы рублей маркетингового бюджета на каждый проводимый эксперимент, повышая общий ROI продуктовой аналитики.
